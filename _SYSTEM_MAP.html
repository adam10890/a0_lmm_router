a0_lmm_router — מפת מערכת

פלאגין שלם של Agent Zero: ניהול שרתי llama.cpp (מקומי/Docker/רימוט) + Smart Router שמנתב הודעות. המסמך הזה ממפה את כל הרכיבים, את הזרימה בין הרכיבים, את הפורטים, ואת נקודות ההדלקה/הכיבוי של כל חלק.

Backend: Python (יושב בתוך A0 container) Frontend: Alpine.js (WebUI של A0) llama.cpp: Docker containers על ה-host גרסה: 1.0.0
Host (Windows) A0 Container Plugin code External services

1. סיכום ב-30 שניות

מה הפלאגין עושה

מנהל שרתי llama.cpp (כל שרת = "slot" ב-port משלו) — מדליק, מכבה, מנטר. ב-Agent Zero הוא מופיע כ-provider אחד או יותר שאפשר לבחור בהם כ-chat_model / utility_model. בנוסף יש Smart Router שמזהה הודעות משתמש לפי patterns ובוחר workflow/slot מתאימים.

החזון שלך (כפי שאני מבין אותו)

מערכת עצמאית ומושלמת — Backend שיודע לבדו להרים ולתחזק את כל ה-containers של llama.cpp, LMM שיודע לנהל את הטרמינלים השונים (chat/utility/embedding/vision/reasoning), ו-Smart Router שמשמש כ-פרונט-אנד — מציג את המצב, מאפשר שליטה, ובוחר ל-A0 איזה slot להשתמש בכל רגע.

7. Smart Router — המצב הנוכחי

⚠️ פער מרכזי לחזון

כרגע ה-Smart Router הוא pattern matcher בלבד. הוא לא בוחר slot באופן דינמי — רק שם workflow_id במטאדטה. בחירת ה-model ל-A0 (chat_model, utility_model) עדיין נעשית דרך Settings הרגיל של A0. כדי להגשים את החזון ("smart router כפרונט-אנד") צריך להרחיב אותו: בחירת provider/model דינמית בכל הודעה.

נספח א׳ — ניתוח Tiny Router והשראה ל-Smart LLM Router

ניתוח מבני של הפלאגין tiny_router (נמצא ב-usr/plugins/tiny_router/) וכיצד ניתן לאמץ את הרעיונות שלו לשיפור ה-Smart LLM Router שלך.

מהו Tiny Router?

פלאגין ל-Agent Zero שמסווג כל הודעת משתמש באמצעות מודל DeBERTa-v3-small (44M פרמטרים, 172MB INT8 ONNX) בזמן של <10ms על CPU. המטרה: חיסכון של 30-50% בעלויות API על ידי:

  • דילוג על קריאות LLM לחלוטין עבור "canned responses" (confirmation, closure)
  • ניתוב אוטומטי ל-model presets זולים יותר עבור הודעות פשוטות
  • שימוש ב-chain של presets ל-failover אוטומטי כשמודל נכשל

א. ארכיטקטורת Tiny Router — רכיבים מרכזיים

1. Extension Hook
_20_tiny_router.py
extensions/python/message_loop_start/
רץ לפני כל הודעת משתמש (hook message_loop_start). מחלץ טקסט, מסווג, מנתב, ומחיל preset override דרך agent.context.set_data().
Lazy Loading
singleton pattern
המודל וה-router נטענים רק בהודעה הראשונה — לא בעליית A0.
2. Classification Engine
DeBERTa-v3-small
tiny_router_helpers/model.py
ONNX Runtime + INT8 quantization. ארבעה "heads":
• relation_to_previous (new/follow_up/correction/confirmation/cancellation/closure)
• actionability (none/review/act)
• retention (ephemeral/useful/remember)
• urgency (low/medium/high)
Temperature Calibration
temperature_scaling.json
כל head מכוון עם temperature נפרד לדיוק confidence.
3. Decision & Routing
router.py
7-rule decision tree
ממיר classification ל-RouteAction: CANNED | LIGHTWEIGHT | CONTEXT_HINT | STANDARD | URGENT
preset_bridge.py
מיפוי ל-A0 presets
ממיר RouteAction → preset name מתוך _model_config של A0.
failover.py

מאפיין Tiny Router a0_lmm_router Smart Router

2. Stats recording (מ-stats.py) 
import json
from pathlib import Path

def record_routing(route: str, slot_url: str, inference_ms: float):
    stats_path = Path("usr/plugins/a0_lmm_router/data/routing_stats.json")
    stats_path.parent.mkdir(parents=True, exist_ok=True)
    
    stats = json.loads(stats_path.read_text()) if stats_path.exists() else {
        "total": 0, "routes": {}, "avg_inference_ms": 0
    }
    
    stats["total"] += 1
    stats["routes"][route] = stats["routes"].get(route, 0) + 1
    # Weighted moving average
    stats["avg_inference_ms"] = (
        stats["avg_inference_ms"] * (stats["total"] - 1) + inference_ms
    ) / stats["total"]
    
    stats_path.write_text(json.dumps(stats, indent=2))
3. Failover detection (מ-failover.py) 
FAILOVER_STATUSES = {408, 429, 500, 502, 503, 504}

def should_failover_to_next_slot(exc: Exception) -> bool:
    """Check if error warrants trying next slot in chain."""
    status = getattr(exc, "status_code", None)
    if status in FAILOVER_STATUSES:
        return True
    
    text = str(exc).lower()
    return any(marker in text for marker in [
        "timeout", "rate limit", "quota exceeded", "connection error"
    ])
